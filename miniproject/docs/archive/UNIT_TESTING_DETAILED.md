# 1.1.1 Unit Testing

Each module of the FutureMatrix AI-powered career guidance platform was tested separately to ensure proper functionality and data accuracy. Unit testing formed the foundation of our comprehensive testing strategy, allowing developers and QA engineers to verify that individual components worked correctly in isolation before attempting to integrate them with other system components.

## Login Module

The Login Module underwent rigorous testing to ensure secure authentication and correct system access control for both student and professional users. The module was tested for student logins using valid email addresses and correct passwords to verify that authenticated users were successfully logged in and correctly redirected to their personalized student dashboard where they could access career prediction, roadmap generation, college discovery, resume analyzer, and other features. Testing also included admin logins with authorized administrator credentials to confirm that admins received appropriate elevated permissions and were redirected to the comprehensive admin panel with analytics, user management, and content management capabilities. Invalid credential testing was equally thorough, including attempts to log in with incorrect passwords, non-existent email addresses, expired or revoked tokens, and malformed input to verify that the system rejected unauthorized access attempts with appropriate error messages without exposing sensitive information that could be exploited by attackers. The OAuth authentication flow with Google Sign-In was tested to ensure seamless integration with Google's authentication service, proper handling of authentication tokens, and correct user session creation. Password reset functionality was tested to verify that users could successfully request password reset emails, receive them within acceptable timeframes, and use reset links to create new passwords. Session management was tested to ensure that authenticated sessions persisted correctly as users navigated between different features, that session tokens were securely stored and transmitted, and that sessions were properly terminated when users logged out or when security policies dictated termination.

## Profile Management Module

The Profile Management Module was tested extensively to verify that users could successfully add, update, and delete their profile information across all seven comprehensive sections including Basic Information (name, email, phone number, date of birth, gender), Education Background (school name, college name, degree earned, graduation year, GPA or percentage), Skills and Competencies (technical skills like programming languages and frameworks, soft skills like communication and leadership, proficiency levels for each skill), Interests and Goals (career interests, learning objectives, hobby interests), Work Experience and Projects (previous job titles, companies, descriptions of responsibilities, academic and personal projects), Social Links and Portfolio (LinkedIn profile URL, GitHub repository, personal portfolio website, other professional profiles), and Personal Preferences (communication preferences, learning style preferences, notification settings). Testing verified that profile information could be successfully added to empty profiles during user onboarding, that existing profile information could be updated after initial creation with changes reflected immediately in the system, and that users could delete specific profile sections or entire records if desired. File upload functionality was extensively tested, particularly for user avatar image uploads, to verify that files were properly uploaded to Supabase Storage with secure file paths and access controls, that only valid image formats (JPEG, PNG, WebP) were accepted while unsupported formats were rejected, that file size limits were enforced to prevent abuse of storage resources, and that uploaded images were associated with the correct user accounts. Validation testing included comprehensive checks for duplicate entries across the system, verification that all required fields were properly validated before data was committed to the database, regex validation of email addresses to ensure proper format, phone number validation for correct format and length, URL validation for social media links to ensure they pointed to valid websites, and handling of edge cases such as very long names, special Unicode characters in international names, and unusual character combinations. Data synchronization testing verified that when profile information was updated in one section, that updated information became immediately available to other modules that relied on profile data for personalization, such as the career prediction engine using education level and skills, or the roadmap generator using selected career interests.

## Career Prediction Module

The Career Prediction Module, being one of the core features of FutureMatrix, received particularly extensive unit testing to ensure accuracy and reliability of the AI-powered career matching system. The comprehensive assessment quiz with 50+ questions was tested with various response patterns to simulate different user types, career interests, and educational backgrounds, ensuring that the system correctly recorded all responses, properly calculated compatibility scores based on the assessment model, and generated appropriate career recommendations specific to each user's profile. Integration with external AI services including Google Gemini and OpenAI GPT-4o was thoroughly tested to verify that the system could successfully communicate with these services, properly format requests with user data and quiz responses, receive valid responses from the APIs within acceptable time limits typically under ten seconds, parse AI-generated responses correctly to extract career recommendations and explanations, handle scenarios where AI services were temporarily unavailable by implementing fallback providers, and recover gracefully from various types of API errors. The calculation of compatibility scores ranging from 0 to 100 was tested against known benchmarks and manually calculated expected outputs to verify mathematical accuracy and consistency across different user profiles. Career recommendation ranking was tested to ensure that top recommended careers were ranked correctly by their compatibility scores in descending order, that the system properly identified the best-matching careers rather than randomly ordering them or using incorrect sorting logic, and that explanations provided by AI for why a particular career was recommended were accurate, relevant, and insightful. Error handling testing included comprehensive scenarios such as users submitting incomplete quiz responses without answering all questions, API calls to external AI services timing out or failing, the system encountering unexpected response formats from APIs that didn't match expected data structures, users attempting to submit quizzes without selecting any answers or meeting minimum response requirements, and network connectivity failures preventing communication with external services. Database persistence testing verified that prediction results were correctly saved with all necessary metadata including the quiz completion date, user ID, complete response data, generated recommendations list, calculated compatibility scores for each career, and AI-generated explanations, and that these results could be accurately retrieved at a later time for display on the user dashboard, in reports, or for use by other modules like roadmap generation.

## Career Roadmap Module

The Career Roadmap Module was tested to verify that the system could generate detailed step-by-step learning progression paths for selected career goals with logical and appropriate phase organization from Foundation level for beginners through Intermediate level for developing competency to Advanced level for expert-level knowledge and Expert level for mastery and specialization. Testing verified that each phase contained appropriate learning objectives, that prerequisites were correctly identified so that subsequent phases could build logically on previously learned skills and knowledge, and that skill progression within each phase proceeded from foundational concepts to more sophisticated applications. Milestone tracking was tested to verify that the system correctly identified meaningful checkpoints in the learning journey, that milestones were meaningfully associated with appropriate learning resources and skill assessments, that users could mark milestones as complete to track their progress through the roadmap, and that completion of earlier milestones was recorded correctly in the database for progress analytics. Time estimation calculations were tested to verify that the system calculated reasonable and achievable time estimates for each phase based on difficulty level and prerequisite skills, that cumulative time estimates for the entire roadmap were accurate and realistic, and that different customization options including accelerated pace for motivated learners, standard pace for typical learners, and flexible pace for learners with limited time could correctly adjust time estimates accordingly. Resource recommendation linking was tested to ensure that each roadmap phase was correctly linked to relevant resources from the learning resources hub, that recommendations matched the specific skills being taught in that phase, and that resource links were accessible and led to valid external educational content. Database persistence testing verified that roadmap data was correctly saved to the PostgreSQL database with all phase information, skill requirements, timeline estimates, and resource links, that saved roadmaps could be retrieved and displayed correctly to users, and that users could save custom modifications to roadmaps and have those modifications persist across sessions.

## College Discovery Module

The College Discovery Module underwent extensive testing to validate the accuracy and comprehensive functionality of the college database and search capabilities essential for students planning their education. Testing verified that the system could correctly store and retrieve detailed information for 1,500+ engineering colleges across India including essential data such as college name and full official name, geographic location including city, state, and coordinates, college type classification as government or private institution, undergraduate and postgraduate courses offered by each college, admission criteria including entrance exam requirements and cutoff scores, placement statistics showing percentage of students placed and average salary, admission fees and total cost of education, infrastructure details like campus size and facilities, national and international rankings, and contact information including official website and admission office details. Searching functionality was tested with various query patterns including full college name searches to find specific known colleges, partial name searches to find colleges by partial matching, and searches using common abbreviations to accommodate users' informal naming preferences, ensuring that the search algorithm correctly identified matching colleges while minimizing false positives that could confuse users. Filtering functionality was tested with each individual filter dimension including state filter to find colleges in specific geographic regions, college type filter to distinguish between government and private institutions, course specialization filters to find colleges offering desired degree programs, rating range filters, fees range filters from affordable to expensive institutions, and cutoff score filters for entrance examinations, as well as testing combinations of multiple filters simultaneously to verify that filtering logic correctly applied multiple constraints and returned accurate results. Advanced filtering testing included scenarios where users applied contradictory filters such as looking for colleges in state X with a specific course that isn't offered in that state, to ensure the system handled such edge cases gracefully by either displaying appropriate error messages or returning empty result sets with helpful suggestions for refining search criteria. The interactive map functionality powered by React Leaflet was tested to verify that college locations were correctly plotted as markers on the map with accurate geographic coordinates, that clustering functionality properly grouped nearby colleges when the map was zoomed out to avoid visual clutter, that zoom and pan operations functioned smoothly without lag or visual artifacts, and that clicking on map markers correctly displayed detailed college information. Pagination functionality was tested to ensure that the system efficiently handled the display of 1,500+ colleges by implementing proper pagination controls, that users could navigate through pages without performance degradation even when accessing later pages, that sorting options including by rating, fees, distance, and placement percentage worked correctly in combination with pagination to maintain sort order across pages, and that page size options allowed users to choose how many colleges to display per page. Save and wishlist functionality was tested to verify that users could successfully bookmark colleges of interest, that bookmarked colleges persisted correctly in the database associated with the user's account, that users could view their saved college list at any time, and that the saved list could be used for side-by-side comparison and informed decision-making about college applications.

## Learning Resources Module

Unit testing of the Learning Resources Module verified the proper display, organization, and functionality of the curated collection of 50+ learning resources including online courses, video tutorials, interactive textbooks, and project-based learning materials across diverse technology domains such as programming fundamentals and advanced techniques, data science and machine learning, artificial intelligence and large language models, web development both frontend and backend, mobile application development, cloud computing and DevOps, and emerging technologies like blockchain and quantum computing. Testing verified that resources were correctly categorized into logical groups and that categorization remained consistent across the system when resources were displayed in different contexts. Filtering functionality was comprehensively tested with filters for skill level to distinguish between beginner, intermediate, and advanced resources, learning format to distinguish between video courses, interactive tutorials, written articles, textbooks, project-based learning, and documentation, estimated duration to help learners choose resources matching their available time, and cost indication to distinguish between free and paid resources. Searching was tested using semantic search capabilities powered by natural language processing to verify that searching for a high-level concept like "machine learning" correctly identified resources about that topic even when the exact phrase wasn't present in the resource title or description, that the search algorithm correctly ranked results by relevance placing most relevant resources first, and that searching worked with synonym variations so searching for "artificial intelligence" and "AI" returned similar results. The AI-powered recommendation engine was tested to verify that it correctly suggested resources based on multiple user factors including the user's career predictions identifying which field to learn, identified skill gaps from resume analyzer showing what specifically to learn, learning preferences from profile indicating preferred learning style, and progress through active roadmaps indicating what should be learned next. Testing ensured that recommended resources were appropriate for the user's current skill level and directly addressed their learning goals. Resource accessibility testing verified that resource links were syntactically correct, that links led to valid external educational platforms without broken links or redirects to irrelevant content, and that clicking links successfully navigated users to the external content without errors or page loading failures. Resource completion tracking was tested to ensure that the system could accurately record which resources users had accessed or completed, that tracking data was correctly stored in the database with timestamps and progress indicators, and that users could view their learning progress over time to see which resources they had completed and which remained to be explored.

## Resume Analyzer Module

The Resume Analyzer Module underwent rigorous testing to validate its core functionality of parsing PDF resumes and generating intelligent analysis using AI services. File upload functionality was comprehensively tested with various file formats including PDF files which are the primary format, DOCX files from Microsoft Word, and plain text files to verify that supported formats were accepted while unsupported formats like image files or spreadsheets were properly rejected with clear error messages. Testing verified that unsupported files were rejected before processing to avoid wasting system resources, that file size limits were enforced to prevent abuse of storage resources, and that upload error messages were clear and helpful to guide users toward acceptable formats. PDF parsing functionality was extensively tested with resumes of varying complexity including resumes with conventional single-column layouts and multiple-column layouts, resumes with embedded images and graphics that might confuse text extraction, resumes containing tables for organizing information, and resumes with special characters and formatting to verify that text extraction worked correctly across different resume styles and that important information wasn't lost during parsing. Testing included deliberately malformed PDF files to verify that the parser handled them gracefully without crashing. Integration with Google Gemini API for AI-powered analysis was tested to verify that the system could successfully send parsed resume content to the Gemini service, receive AI-generated analysis within acceptable time limits, and handle scenarios where the Gemini service was temporarily unavailable by switching to alternative AI providers like OpenAI or Cohere. ATS (Applicant Tracking System) score calculation was tested by comparing the calculated scores against known benchmarks and manual expert evaluations of the same resumes to ensure accuracy. Skill extraction testing verified that the system could identify and correctly extract technical skills including specific programming languages like Python, JavaScript, Java and frameworks like React, Django, Spring, soft skills including communication, leadership, teamwork, and domain expertise in specific industries. Gap identification testing verified that the system could identify missing skills that are commonly required for the careers that matched the user's profile by comparing extracted skills against job requirement databases, and could suggest learning resources to fill these gaps. Improvement suggestions testing verified that suggestions were actionable and specific rather than generic, relevant to the user's particular situation and career goals, and actually achievable within reasonable timeframes without requiring years of additional education. Interview question generation was tested to ensure that generated interview questions were relevant to the user's actual background and experience, were appropriately challenging for the career level indicated in the resume, and covered common interview topics while avoiding generic questions that anyone might be asked. Error handling tested comprehensive scenarios including corrupted PDF files that couldn't be parsed correctly, resumes with unrecognizable content that didn't appear to be resume data, completely empty PDF files containing no text content, extremely long resumes exceeding typical length, and files with restricted permissions that couldn't be accessed, with verification that the system failed gracefully and provided helpful error messages in all cases.

## Job Hunting Module

The Job Hunting Module unit tests verified that the system could correctly identify, match, and display job opportunities relevant to user profiles and career goals. Job matching algorithm testing verified that jobs were matched based on multiple relevant factors including skills extracted from resume or profile, experience level indicated by work history, education requirements and degree specialization, location preferences or willingness to relocate, and alignment with career goals identified by career predictions. Testing included verification that job recommendations changed appropriately when profile information was updated, that the matching algorithm correctly weighted different matching factors so more important factors had greater influence, and that highly relevant jobs were ranked higher in result listings than less relevant opportunities. Job listing display testing verified that all important job information was correctly displayed including job title and full description, hiring company name and industry, geographic location or remote work indicator, salary range if disclosed, job type classification as full-time, part-time, contract, or internship, required qualifications and experience level, and application deadline or submission link. Filtering functionality was extensively tested with filters for geographic location to find jobs in preferred regions, job type to distinguish between different employment arrangements, experience level from entry-level to senior positions, industry or job category, salary range to find jobs matching financial expectations, and company size to find opportunities in companies of preferred size. Testing included scenarios where filters returned no results to ensure appropriate messages were displayed and suggestions were provided for refining search criteria. Job save and bookmark functionality was tested to ensure users could save interesting job opportunities they wanted to consider later, that saved jobs persisted correctly in the database associated with the user account, that users could access their saved job list at any time, and that saved jobs could be used to compare opportunities and make informed application decisions. Job alert functionality was tested to verify that users could configure alert criteria based on their job preferences, that the system correctly identified new job postings matching alert criteria, that notifications were sent to users at appropriate intervals avoiding alert fatigue from too-frequent notifications, and that users could view and manage their configured job alerts.

## Admin Panel Module

Admin Panel testing verified all administrative functions for platform management, monitoring, and analytics. User management testing verified that authorized admins could view comprehensive lists of all registered users, access detailed user profiles including all profile information, view complete user activity history and engagement metrics, temporarily deactivate user accounts when necessary, reset user passwords to help users regain access if forgotten, manage user roles and permission levels to assign appropriate responsibilities, and view user subscription tier and payment status. Testing verified that these administrative operations were properly logged in the audit trail for security compliance and that the system required confirmation for destructive operations to prevent accidental data loss. College database management testing verified that admins could successfully add new colleges to the database with all relevant information, edit existing college data to update outdated information, delete colleges from the database when they closed or became irrelevant, and perform bulk operations to simultaneously add or update multiple colleges from data files to simplify database maintenance. Learning resource management testing verified similar CRUD (Create, Read, Update, Delete) operations for maintaining the resource library and ensuring content remained current and relevant. Analytics dashboard testing verified that the dashboard correctly displayed and calculated key metrics including total number of registered users showing platform growth, user signup trends over time indicating adoption patterns, career prediction conversion rates showing what percentage of users completed the assessment, most popular careers recommended indicating where student interests lay, college search frequency indicating student planning activity, and user engagement metrics like average session duration and features used. Chart rendering was tested to ensure that graphs and visualizations displayed correctly with proper axes and labels, that data points were accurately plotted without errors or omissions, and that charts remained interactive and responsive to user interactions like hovering for details or filtering. Promo code management testing verified that admins could create new promotional codes with specified discount amounts and usage limits, edit existing codes to adjust parameters as campaigns evolved, view comprehensive usage statistics for each promo code including how many were used and by whom, and deactivate codes when promotional periods ended. Subscription tier management testing verified that admins could modify tier benefits like feature access and limits, adjust pricing for different tiers, and that changes to subscription tiers properly affected which features were available to users in each tier.

## Tools Used

The FutureMatrix system was tested using a comprehensive suite of testing tools and frameworks appropriate for a modern full-stack JavaScript/TypeScript web application. Playwright, a modern end-to-end testing framework developed by Microsoft, was used for automated testing of critical user workflows and interactions across different browsers including Chrome, Firefox, and Safari and across different device types including desktop, tablet, and mobile to verify that the user interface worked correctly and that user actions consistently produced expected results. Jest and Vitest, popular and powerful JavaScript testing frameworks, were used for unit testing React components in isolation to verify that components rendered correctly with different prop combinations and responded appropriately to user interactions, testing pure utility functions and helper functions to ensure business logic worked correctly independent of the React rendering layer, testing API route handlers to verify endpoint functionality before integration with the full application, and testing database query functions to ensure data access logic worked correctly. Postman, a comprehensive API testing platform, was used to manually and automatically test backend API endpoints in isolation by sending requests with various parameters and input combinations, verifying that endpoints accepted correct input and rejected invalid input, confirmed that endpoints processed data correctly according to business logic, and returned expected response formats and HTTP status codes. Thunder Client provided an alternative API testing tool for quick API verification during development. Manual testing was conducted by experienced developers and QA engineers by running the application in VS Code development environment with hot module reloading for rapid iteration, using modern web browsers to interact with the application like real end users would, clicking buttons to trigger actions, filling out forms with various data values, navigating between pages to verify routing, and systematically verifying that functionality worked as expected. Database testing was performed using the Supabase console with graphical query builders and SQL editors to directly query the PostgreSQL database and verify that data was correctly stored in appropriate tables with correct data types and values, pgAdmin the comprehensive PostgreSQL administration tool to perform more advanced database analysis including index verification and query performance analysis, and direct SQL queries to perform complex validation checks on data consistency and referential integrity. This comprehensive multi-level testing approach combining automated testing with manual testing by human testers ensured that individual components worked correctly in isolation while also catching issues that automated tests might miss and verifying the overall user experience.
